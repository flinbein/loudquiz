# Глоссарий

Единый справочник терминов, используемых в документации Loud Quiz.
Термины на английском языке используются в коде, интерфейсах и формулах.

## Роли

| Термин | Описание |
|---|---|
| host | Ведущий. Управляет игрой с большого экрана |
| player | Игрок. Подключается с мобильного устройства |
| spectator | Зритель. Наблюдает за игрой на главном экране, не взаимодействует с приложением |
| captain | Капитан раунда. Видит задание, объясняет его жестами и мимикой, и сам тоже отвечает на вопрос |
| responders | В блиц-раунде: игроки команды без капитана, принимающие объяснение по цепочке. В обычном раунде отвечают **все** члены команды, включая капитана |

## Команды

| Термин | Описание |
|---|---|
| active team | Команда, играющая текущий раунд |
| opponent team | Команда-соперник (только в режиме dual) |
| team_size | Общее количество игроков в команде, **включая капитана** |
| responders_count | Количество responders = `team_size - 1` (используется только в блице) |

### Ограничения

- Минимум **2** игрока в команде (капитан + 1 responder).
- Максимум **10** игроков в команде.

## Раунды

| Термин | Описание |
|---|---|
| round | Обычный раунд. Капитан объясняет задание жестами всем responders одновременно. Каждый responder даёт свой ответ |
| blitz | Блиц-раунд. Капитан объясняет слово первому игроку, тот — следующему, и так по цепочке до последнего |
| chain | Цепочка объяснений в блице: капитан → игрок 1 → игрок 2 → ... → игрок N |
| chain_length | Количество передач в цепочке = `responders_count` |

## Игровые элементы

| Термин | Описание |
|---|---|
| joker | Карта джокера. Удваивает очки за раунд (×2). Одна на команду за всю игру |
| bonus time | Бонусное время. Оставшееся время таймера, если все ответили досрочно |
| difficulty | Сложность задания. 100–200 для обычных раундов, 200–400 для блица. Кратно 10 |

## Режимы игры

| Термин | Описание |
|---|---|
| single mode | Режим одной команды. Кооперативный — цель набрать максимум баллов |
| dual mode | Режим двух команд. Соревновательный — команды играют по очереди |
| manual mode | Ручной режим. Вопросы загружаются из JSON-файла, ведущий оценивает ответы |
| ai mode | Режим AI. Вопросы генерируются через OpenRouter API, AI оценивает ответы |

## Использование в формулах

В формулах подсчёта очков и таймеров используются следующие обозначения:

- `team_size` — все игроки команды, включая капитана
- `responders_count` — `team_size - 1` (без капитана)
- `номер_игрока` — порядковый номер в цепочке блица, начиная с 1 после капитана

Примеры:
- Команда из 4 человек: `team_size = 4`, `responders_count = 3`
- Таймер round-active: `55с + 5с × team_size` (капитан тоже отвечает)
- Таймер blitz-active: `10с + 20с × responders_count`
