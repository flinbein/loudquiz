# AI-интеграция

Интеграция с AI через OpenRouter API для генерации заданий
и автоматической проверки ответов.

## OpenRouter API

Все AI-операции выполняются через [OpenRouter](https://openrouter.ai/) —
агрегатор LLM-моделей с единым API.

### Аутентификация

API-ключ указывается пользователем:
- В **редакторе вопросов** — сохраняется в localStorage.
- При **создании игры** в AI-режиме — вводится в настройках.

Ключ хранится в `localStorage` и никогда не передаётся
другим участникам через транспорт.

### Модель

Рекомендуемая модель: `google/gemini-2.0-flash-001`.
Модель должна поддерживать structured output (JSON schema).

## Формат запросов

Все запросы к AI используют chat completions API с ролями:

| Роль | Назначение |
|---|---|
| `system` | Инструкции и правила для AI |
| `user` | Входные данные (темы, вопросы, ответы) |

Ответ запрашивается в формате **JSON** с заданной схемой
(structured output / response_format).

## Обработка ошибок

### Общий паттерн

При любой ошибке AI (сеть, таймаут, невалидный ответ):
текст ошибки + кнопка **[Повторить]**.

При невалидном JSON в ответе — автоматический повтор запроса (до 3 попыток),
затем отображение ошибки пользователю.

### topics-suggest

Ошибка генерации тем или вопросов → сообщение об ошибке + **[Повторить]**.

### round-review

Ошибка проверки ответов → сообщение об ошибке + **[Повторить]** + **[Оспорить]**.
Кнопка «Оспорить» переводит в ручной режим оценки (host оценивает ответы вручную).
Это гарантирует, что игра не заблокируется при недоступности AI.
