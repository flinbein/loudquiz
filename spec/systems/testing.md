# Тестирование

Подходы к тестированию Loud Quiz и доступ к состоянию
для автоматизированных тестов.

## Инструменты

| Инструмент | Назначение |
|---|---|
| Vitest | Юнит-тесты: игровая логика, подсчёт очков, утилиты |
| Playwright | E2E-тесты, скриншот-тесты |
| Ladle | Визуальное тестирование компонентов |

## Доступ к store из тестов

В dev-режиме Zustand store экспортируется в глобальный объект:

```typescript
// В приложении (dev-режим)
if (import.meta.env.DEV) {
  window.__store = useGameStore;
}
```

### Использование в Playwright

```typescript
// Чтение состояния
const phase = await page.evaluate(() => {
  return window.__store.getState().phase;
});

// Установка состояния (мок)
await page.evaluate(() => {
  window.__store.setState({
    phase: "round-active",
    players: [/* ... */],
    // ...
  });
});
```

Это позволяет:

- **Мокать** любое состояние игры для скриншот-тестов.
- **Проверять** переходы между фазами.
- **Изолировать** UI-тесты от транспортного слоя.

## Скриншот-тесты

Для каждой фазы игры создаётся набор скриншот-тестов:

1. Установить состояние через `window.__store.setState(...)`.
2. Дождаться рендера.
3. Сделать скриншот и сравнить с эталоном.

Покрытие:
- Каждая фаза для host и player.
- Различные состояния компонентов (открытый/закрытый конверт, с джокером и т.д.).
- Светлая и тёмная тема.

## Broadcast Channel для E2E

Для E2E-тестов используется [Broadcast Channel](transport/broadcast-channel.md)
как транспорт — позволяет запускать host и player в одном браузере.
