# VarHub

Реализация [транспортного слоя](README.md) на основе WebSocket
с использованием [VarHub](https://www.npmjs.com/package/@flinbein/varhub-web-client).

## Назначение

Связь между устройствами через WebSocket-сервер VarHub.
Обеспечивает более надёжное и быстрое соединение по сравнению с p2pt.

## Принцип работы

1. Host создаёт комнату на VarHub-сервере (ws-тип).
2. Player подключается к комнате по roomId.
3. Обмен сообщениями идёт через сервер.

## Особенности

- **Требует сервер** — нужен работающий VarHub-инстанс.
- **Быстрое подключение** — WebSocket устанавливается мгновенно.
- **Надёжность** — не зависит от NAT и конфигурации сети.
- **Таймаут** — комната может быть закрыта сервером, если
  в ней нет активных участников в течение ~2–5 минут.

## Приоритет реализации

**Третий** — реализуется после p2pt.
