# Работа с AI

Использовать api openrouter.

Для отладки OPENROUTER_API_KEY записан в файле `.env` в корне проекта. 

Нужны следующие операции:

## Генерация тем для викторины

Входные данные:

```typescript
interface Input {
  suggestions: Array<{
    playerName: string
    text: string
  }>;
  topicCount: number;
  pastTopics: string[];
}
```

Работа:

- ИИ должен выбрать {N} тем для викторины, учитывая пожелания большинства игроков.
- ИИ должен игнорировать непонятные или нецензурные предложения.
- ИИ не должен выбирать узкоспециализированные темы.
- ИИ не должен выбирать темы, которые уже участвовали в игре.
- Если предложений мало, ИИ может предложить свои варианты тем.

Результат:

```typescript
interface Result {
  topics: Array<{
    name: string;   // название темы 
    reason: string; // причина, почему выбрал эту тему
  }>
}
```

## Генерация вопросов для викторины

Входные данные:

```typescript
interface Input {
  topics: string[];
  questionsPerTopic: number,
  playersPerTeam: number,
  pastQuestions: string[],
}
```

Работа:

- Каждый вопрос должен иметь **не менее `playersPerTeam` уникальных правильных ответов**. Синонимы или похожие слова не являются уникальными.
- Ответы должны быть **проверяемыми** (ведущий или ИИ должны иметь возможность подтвердить)
- Сложность задаётся явно (100–200, кратно 10) и определяется:
  - Насколько сложно **передать смысл жестами и мимикой** (основной критерий)
  - Длинная формулировка → сложнее
  - **Исключающие условия** ("Млекопитающее из трёх слогов", "Месяц с 31 днём") → сложнее
  - **Каламбур или загадка** ("Город в стране восходящего солнца") → сложнее
- Вопросы не должны повторять `pastQuestions`
- Распределение сложности в теме: примерно равномерно от 100 до 200
- `acceptedAnswers` — `playersPerTeam` примеров правильных ответов.

Результат:

```typescript
interface Result {
  topics: Array<{
    "name": string;                     // название темы
    "questions": Array<{
      "text": string,                 // текст вопроса
      "difficulty": 200,            
      "acceptedAnswers": string       // примеры ответов
    }>
  }>
}
```

## Проверка ответов и комментарий к раунду

Входные данные:

```typescript
interface Input {
  question: string // текст вопроса
  answers: Array<{ 
      playerName: string; 
      answer: string; 
  }>;
}
```

Работа:
ИИ проверяет ответы и группирует их при необходимости
- ✅ Ответ **правильный**: соответствует вопросу, поддаётся проверке
- ❌ Ответ **расплывчатый**: нельзя проверить, является ли он корректным ("Я", "Какой-то космонавт")
- ❌ Ответ **одинаковый**: семантически совпадает с другим → объединяются в группу, считается за 1 уникальный ответ
  - Исключение: если вопрос про синонимы/похожие варианты — оба принимаются
- ✅ Небольшие опечатки допустимы, если вопрос не требует точного написания
ИИ пишет короткий итог раунда.

Результат:

```typescript
interface Result {
  results: Array<{
    playerName: string
    accepted: boolean    // ответ верный или нет
    group: number | null // номер группы, если ответ объединён
    note: string | null  // комментарий, почему отет засчитан, не засчитан, или объединён.
  }>;
  comment: string        // общий итог раунда
}
```

## 9.4 Генерация заданий для блиц

**Входные данные:**
```typescript
interface Input {
  rounds: number         // количество раундов
  tasksPerRound: number  // количество вариантов заданий на раунд
  pastTasks: string[]    // прошлый задания, которые нельзя использовать
}
```

Правила:
- Каждое задание — набор слов/словосочетаний разной сложности, которые капитан объясняет **жестами и мимикой**
- Сложность определяется тем, насколько сложно изобразить слово без звука:
  - 200 — легко показать (фрукт, животное, простое действие)
  - 300 — умеренно (профессия, предмет быта, известная личность)
  - 400 — сложно (абстрактное понятие, малоизвестное слово, составное понятие)
- Абстрактные понятия ("Счастье", "Демократия") допустимы, но относятся к высокой сложности (≥ 350)
- `text` одновременно является заданием и правильным ответом; нормализованная форма для сравнения вычисляется автоматически (нижний регистр, `ё` → `е`)
- Не повторять слова из `pastTasks`
- Внутри каждого задания варианты должны охватывать разные уровни сложности

Результат:

```typescript
interface Result {
  rounds: Array<{
    items: Array<{
      text: string;        // задание: слово или словосочетание
      difficulty: number;  // Сложность задания: 200 - 400
    }>;
  }>;
}
```
