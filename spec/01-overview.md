# Обзор проекта

Loud Quiz — командная викторина для вечеринок. Игроки надевают наушники с музыкой
и не слышат друг друга. Капитан команды видит вопрос и объясняет его жестами и мимикой.
Каждый игрок должен самостоятельно понять вопрос и дать уникальный ответ.

## Концепция

Игра проводится на большом экране (host), который виден всем участникам.
Каждый игрок подключается со своего мобильного устройства через QR-код или ссылку.
Ведущий управляет ходом игры с главного экрана.

Во время активных раундов игроки слушают музыку в наушниках, что создаёт
барьер коммуникации — единственный способ передать информацию — жесты и мимика.

## Роли

### Host (ведущий)

- Запускает игру на большом экране
- Управляет ходом игры: переходы между фазами, проверка ответов
- В ручном режиме — оценивает ответы игроков
- В AI-режиме — контролирует оценку AI, может оспорить решение

### Player (игрок)

- Подключается со своего устройства
- Входит в команду, выбирает аватар
- Может стать капитаном раунда
- Вводит ответы на вопросы

### Spectator (зритель)

- Наблюдает за игрой на главном экране
- Не взаимодействует с приложением напрямую

## Режимы игры

### По количеству команд

- **Одна команда** (single mode) — кооперативный режим. Одна команда играет
  все раунды подряд. Цель — набрать максимум баллов. Противников нет.
- **Две команды** (dual mode) — соревновательный режим. Команды играют по очереди.
  Каждая команда должна сыграть **одинаковое** количество обычных и блиц-раундов.
  Если количество заданий нечётное — последнее не играется.
  Пример: 1 блиц-задание на 2 команды — блиц-раунды не проводятся.

### Ограничения

- В команде от **2** до **10** игроков (включая капитана).
- Подробнее о терминах: [глоссарий](game/glossary.md).

### По источнику заданий

- **Ручной режим** — ведущий загружает JSON-файл с подготовленными вопросами.
  Оценку ответов выполняет ведущий вручную.
- **AI-режим** — приложение генерирует вопросы через OpenRouter API.
  Оценку ответов выполняет AI. Игроки могут оспорить решение AI,
  в этом случае ведущий оценивает ответы вручную.

## Общий ход игры

```
lobby → [topics-suggest] → (round | blitz)+ → finale
```

1. **Lobby** — игроки подключаются, выбирают команды, настраивают наушники.
2. **Topics Suggest** (только AI-режим) — игроки предлагают темы, AI выбирает.
3. **Раунды** — чередование обычных раундов и блиц-раундов.
   Сначала играются карточки с заданиями, затем блиц-задания.
4. **Finale** — итоги игры, статистика, возможность начать заново.

## Общие элементы интерфейса

На всех экранах (host и player) доступны:

- **Переключение темы** — светлая / тёмная / системная
- **Полноэкранный режим**
- **Калибровка** — настройка громкости музыки, сигнала, вибрации

Приложение использует **WakeLock API** для предотвращения гашения экрана
во время игры.
