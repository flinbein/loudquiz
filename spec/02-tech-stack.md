# Технологический стек

Loud Quiz — полностью клиентское веб-приложение без собственного бекенда.
Результат сборки — набор статических файлов для размещения на GitHub Pages
или любом другом статическом хостинге.

## Основные технологии

| Технология | Назначение |
|---|---|
| React | UI-фреймворк |
| TypeScript | Типизация |
| Vite | Сборка и dev-сервер |
| CSS Modules | Стилизация компонентов |
| Zustand | State management |
| i18next | Локализация (ru, en) |
| OpenRouter API | Интеграция с AI |

## Транспортный слой

Связь между host и player осуществляется через абстрактный транспортный интерфейс
с тремя реализациями:

| Реализация | Назначение | Приоритет |
|---|---|---|
| Broadcast Channel API | Локальная отладка (одна вкладка = один участник) | Первый |
| p2pt | WebRTC через BitTorrent-трекеры, без сервера | Второй |
| VarHub | WebSocket rooms | Третий |

Выбор реализации — в настройках приложения.

Подробнее: [Транспортный слой](systems/transport/README.md)

## Инструменты разработки

| Инструмент | Назначение |
|---|---|
| Ladle | Библиотека компонентов (Storybook-совместимый) |
| Vitest | Юнит-тесты |

## Команды

```bash
npm run dev            # Dev-сервер, доступен в локальной сети (0.0.0.0)
npm run dev:storybook  # Ladle — библиотека компонентов
npm run build          # Сборка статических файлов
npm run test           # Запуск тестов
```

## Управление состоянием

Zustand store — единый источник состояния игры. Store экспортируется
в `window.__store` в dev-режиме для доступа из Playwright/Puppeteer.

Подробнее: [Управление состоянием](systems/state.md),
[Тестирование](systems/testing.md)

## Локализация

Приложение поддерживает два языка: русский (по умолчанию) и английский.
Используется i18next с JSON-файлами переводов.
Поддерживаются плюральные формы (i18next plural rules).

## Роутинг

| Маршрут | Страница |
|---|---|
| `/` | Главная страница |
| `/constructor` | Редактор вопросов |
| `/rules` | Правила игры |
| `/play?room=<id>` | Активная игра |

### Параметры `/play`

- `room` — единственный параметр. ID комнаты.
- Тип транспорта определяется по **первой букве** room ID:
  `v` = VarHub, `b` = Broadcast Channel, `p` = p2pt.
- URL одинаковый для host и player. Host определяется по наличию
  данных в sessionStorage (см. [персистентность](systems/persistence.md)).
