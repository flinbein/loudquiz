# Персистентность данных

Механизмы сохранения данных между сессиями и при перезагрузке страницы.
Loud Quiz — клиентское приложение без бекенда, поэтому все данные
хранятся в браузере.

## sessionStorage (host)

Host сохраняет [GameState](state.md) в sessionStorage для восстановления
при случайной перезагрузке страницы.

- Записывается при каждом значимом изменении состояния.
- При загрузке `/play` host проверяет наличие сохранённого состояния
  и восстанавливает игру.
- Именно наличие данных в sessionStorage отличает host от player
  при открытии одного и того же URL.

## localStorage

Данные, которые сохраняются между сессиями:

| Ключ | Назначение | Используется |
|---|---|---|
| API-ключ OpenRouter | Аутентификация в AI-операциях | Конструктор, создание игры |
| Имя игрока | Предзаполнение при входе | Lobby (player) |
| Настройки калибровки | Громкость музыки, сигнала, вибрация | Все экраны |
| Использованные вопросы | Исключение повторов при генерации | AI-режим, конструктор |

### Использованные вопросы

При завершении игры в AI-режиме текущие задания добавляются в список
использованных вопросов. При «Играем ещё раз» или при новой генерации
в конструкторе этот список передаётся в AI для исключения повторов.
