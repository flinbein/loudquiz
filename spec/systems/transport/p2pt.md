# p2pt

Реализация [транспортного слоя](README.md) на основе WebRTC
с использованием библиотеки [p2pt](https://www.npmjs.com/package/p2pt).

## Назначение

Связь между устройствами без собственного сервера. В качестве
сигнального сервера используются публичные BitTorrent-трекеры.

## Принцип работы

1. Host создаёт комнату — генерирует уникальный идентификатор (roomId),
   который используется как «torrent identifier» для p2pt.
2. Player подключается по roomId — p2pt находит host через трекеры
   и устанавливает прямое WebRTC-соединение.
3. После установки соединения обмен данными идёт напрямую (peer-to-peer).

## Особенности

- **Без сервера** — не требует собственного бекенда.
- **NAT traversal** — WebRTC использует STUN/TURN для обхода NAT.
- **Задержка подключения** — установка соединения через трекеры
  может занять несколько секунд.

## Приоритет реализации

**Второй** — реализуется после Broadcast Channel.
