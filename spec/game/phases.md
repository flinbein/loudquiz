# Фазы игры

Игра Loud Quiz проходит через последовательность фаз (GamePhase).
Каждая фаза определяет текущее состояние игры, доступные действия
и отображаемые экраны.

## Типы фаз

```typescript
type GamePhase =
  | "lobby"           // подключение игроков, выбор команды
  | "topics-suggest"  // генерация заданий (только AI-режим)
  | RoundPhase        // обычный раунд
  | BlitzPhase        // блиц-раунд
  | "finale"          // конец игры

type RoundPhase =
  | "round-captain"   // выбор капитана
  | "round-pick"      // выбор вопроса
  | "round-ready"     // подготовка, наушники
  | "round-active"    // таймер, объяснение жестами
  | "round-answer"    // ввод ответов (доп. время)
  | "round-review"    // проверка ответов, итоги

type BlitzPhase =
  | "blitz-captain"   // выбор капитана + очерёдность
  | "blitz-pick"      // выбор блиц-задания
  | "blitz-ready"     // подготовка, наушники
  | "blitz-active"    // таймер, цепочка объяснений
  | "blitz-answer"    // ввод ответа (доп. время)
  | "blitz-review"    // результат блица
```

## Общая схема переходов

```
lobby
  │
  ├─ [AI-режим] ──→ topics-suggest
  │                      │
  │                      ▼
  └──────────────→ round-captain ──→ round-pick ──→ round-ready
                                                        │
                       round-review ◄── round-answer ◄──┤
                            │                           │
                            │              round-active ─┘
                            │
                            ▼
                   [ещё карточки?] ──да──→ round-captain
                            │
                           нет
                            │
                            ▼
                   blitz-captain ──→ blitz-pick ──→ blitz-ready
                                                        │
                       blitz-review ◄── blitz-answer ◄──┤
                            │                           │
                            │              blitz-active ─┘
                            │
                            ▼
                   [ещё блиц?] ──да──→ blitz-captain
                            │
                           нет
                            │
                            ▼
                         finale
```

## Чередование команд

### Single mode (одна команда)

Одна команда играет все раунды подряд. Нет чередования, нет противников.

### Dual mode (две команды)

Команды играют **по очереди**. Каждая команда должна сыграть одинаковое
количество обычных и блиц-раундов.

- Если количество обычных заданий **нечётное** — последнее не играется.
- Если количество блиц-заданий **нечётное** — последнее не играется.
- Пример: 1 блиц-задание на 2 команды — блиц-раунды не проводятся.

## Описание фаз

### lobby

Подключение игроков, выбор команд, настройка наушников.

- **Музыка:** нет
- **Таймер:** нет
- **Условие перехода:** все игроки нажали «Готов», ведущий нажимает «Начать»
- **Ограничение:** после начала игры новые игроки не допускаются

Подробнее: [host](../ui/host-screens/lobby.md),
[player](../ui/player-screens/lobby.md)

### topics-suggest (только AI-режим)

Игроки предлагают темы. AI выбирает N тем и генерирует задания.

- **Музыка:** нет
- **Таймер:** 60 секунд (фаза 1)
- **Фаза 1:** игроки вводят темы, AI выбирает
- **Фаза 2:** отображение сгенерированных заданий
- **Условие перехода:** ведущий нажимает «Начать игру»

Подробнее: [host](../ui/host-screens/topics-suggest.md),
[player](../ui/player-screens/topics-suggest.md)

### round-captain

Выбор капитана команды.

- **Музыка:** нет
- **Таймер:** 60 секунд
- **Действие:** игрок нажимает «Я капитан» (первый нажавший становится капитаном)
- **Ограничение:** один игрок не может быть капитаном 2 раза подряд
  в рамках своей команды. Допускается чередование (A, B, A, B, C)
- **Таймаут:** капитан назначается случайным образом

Подробнее: [host](../ui/host-screens/round.md#round-captain),
[player](../ui/player-screens/round.md#round-captain)

### round-pick

Капитан выбирает вопрос из таблицы заданий.

- **Музыка:** нет
- **Таймер:** 60 секунд
- **Действие:** капитан кликает на вопрос в таблице; может активировать джокер
- **Таймаут:** выбирается случайный вопрос

Подробнее: [host](../ui/host-screens/round.md#round-pick),
[player](../ui/player-screens/round.md#round-pick)

### round-ready

Подготовка к раунду. Игроки надевают наушники.

- **Музыка:** включается у игроков активной команды
- **Таймер:** нет
- **Действие:** каждый игрок нажимает «Я готов»
- **Условие перехода:** все игроки активной команды готовы

Подробнее: [host](../ui/host-screens/round.md#round-ready),
[player](../ui/player-screens/round.md#round-ready)

### round-active

Капитан видит вопрос и объясняет жестами. Игроки вводят ответы.

- **Музыка:** играет у игроков активной команды
- **Таймер:** `50с + 5с × responders_count` (см. [глоссарий](glossary.md))
- **Сигнал:** звуковой сигнал + вибрация в начале и конце фазы
- **Условие перехода:**
  - Все responders ответили → `round-review` (бонусное время учитывается)
  - Таймер истёк → `round-answer` (бонусное время = 0)

Подробнее: [host](../ui/host-screens/round.md#round-active),
[player](../ui/player-screens/round.md#round-active)

### round-answer

Дополнительное время для игроков, не успевших ответить.

- **Музыка:** плавно затухает (~3 сек)
- **Таймер:** 20 секунд
- **Условие перехода:** все игроки ответили или таймер истёк → `round-review`

Подробнее: [host](../ui/host-screens/round.md#round-answer),
[player](../ui/player-screens/round.md#round-answer)

### round-review

Проверка ответов и подведение итогов раунда.

- **Музыка:** нет
- **Фаза 1 (ручной):** ведущий оценивает ответы, объединяет одинаковые
- **Фаза 1 (AI):** AI проверяет ответы, отображается лоадер
- **Фаза 2:** результаты, баллы, кнопка «Оспорить» (→ ручная оценка)
- **Условие перехода:** ведущий или игроки нажимают «Продолжить»

Подробнее: [host](../ui/host-screens/round.md#round-review),
[player](../ui/player-screens/round.md#round-review),
[подсчёт очков](scoring.md)

### blitz-captain

Выбор капитана и очерёдности игроков для блица.

Подробнее: [блиц-раунды](blitz.md#blitz-captain)

### blitz-pick

Капитан выбирает блиц-задание.

Подробнее: [блиц-раунды](blitz.md#blitz-pick)

### blitz-ready

Подготовка к блиц-раунду.

Подробнее: [блиц-раунды](blitz.md#blitz-ready)

### blitz-active

Цепочка объяснений между игроками.

Подробнее: [блиц-раунды](blitz.md#blitz-active)

### blitz-answer

Дополнительное время для ответа.

Подробнее: [блиц-раунды](blitz.md#blitz-answer)

### blitz-review

Результат блиц-раунда.

Подробнее: [блиц-раунды](blitz.md#blitz-review)

### finale

Итоги игры.

- **Музыка:** нет
- Отображается счёт, статистика, возможность начать заново (AI-режим)
- **«Играем ещё раз»** (только AI-режим): переход к `lobby` в той же комнате.
  Новые игроки могут присоединиться. Использованные вопросы сохраняются
  в localStorage для исключения при следующей генерации.
  Настройки игры ([GameSettings](../systems/state.md)) сохраняются.

Подробнее: [host](../ui/host-screens/finale.md),
[player](../ui/player-screens/finale.md),
[подсчёт очков](scoring.md#статистика-в-финале)
