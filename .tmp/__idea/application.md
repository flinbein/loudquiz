# Описание приложения

## Главный экран
Ведущий запускает веб-приложение на большом экране, который виден всем.

Кнопки: [Новая игра], [Войти], [Редактор вопросов], [Правила игры]

## Новая игра

Ведущий настраивает параметры для игры

### Командный режим
- Одна команда. (Цель игроков - набрать наибольшее количество баллов)
- Две команды. (Игроки соревнуются)

### Задания
Ведущий должен выбрать одно из действий:
- Загрузить json с вопросами (ручной режим)
- Выбрать режим игры с ИИ (режим AI)

### Предпросмотр (ручной режим)
Ведущий видит задания в виде таблицы. Есть возможность передумать и загрузить другой json.

### Дополнительные настройки (AI режим)
Ведущий указывает:
- API KEY
- Количество тем (по умолчанию 3)
- Количество заданий в теме (по умолчанию 4)
- Количество блиц-раундов на каждую команду (по умолчанию 2)
- список вопросов, которые ранее участвовали в игре.

Проверка: 
```
([количество тем] * [количество заданий]) > 0 || [Количество блиц-раундов] > 0
```

## Подключение к игре

- Игрок переходит по URL (из QR-кода или по ссылке)
- Игрок вводит имя (предзаполнено из localStorage)
- Игрок попадает в игру, подключается к хосту.

Имя игрока считается его уникальным id. Нельзя допускать двух игроков с одинаковым именем: отключить игрока, который зашёл раньше.

Если игрок отключился - его статус в игре меняется на "оффлайн", игрок может переподключиться и продолжить игру.

Как только игра началась - новые игроки не допускаются.

## Игра

В процессе игры для игроков и ведущего всегда доступны кнопки (в углу экрана): Переключение темы (светлая/темная/системная), На полный экран, калибровка.

У каждой команды есть карта джокера. Активировать можно 1 раз за игру в `RoundPhase`. Карта умножает количество очков раунда на 2.

Игроки сначала играют карточки с заданиями. Затем играют блиц-задания.

### Калибровка

Открывается попап со сдеюующим интерфейсом:

- Музыка: Кнопка (play/pause), настройка уровня громкости. Сохранять в localStorage
- Сигнал: Кнопка для воспроизведения сигнала, настройка уровня громкости. Сохранять в localStorage. При воспроизведении сигнала включать вибрацию.
- Вибрация: Переключатель ВКЛ/ВЫКЛ
- Текстовые инструкции.

Игрок должен настроить уровень громкости таким образом, чтобы при воспроизведении музыки не слышать других игроков.
В процессе воспроизведения музыки должен быть различим звуковой сигнал.
Рекомендуется отключить уведомления на мобильном устройстве на время игры.

### Схема состояния игры

```typescript
type GamePhase =
  | "lobby"           // подключение игроков, выбор команды
  | "topics-suggest"  // Генерация списка вопросов (только режим AI)
  | RoundPhase        // раунд стандартный
  | BlitzPhase        // раунд блиц
  | "finale"          // конец игры

type RoundPhase =
  | "round-captain"   // выбор капитана (без музыки)
  | "round-pick"      // таблица вопросов у всех игроков команды; капитан выбирает (без музыки)
  | "round-ready"     // наушники, музыка включается, все нажимают "Я готов" (включая капитана)
  | "round-active"    // таймер идёт, музыка играет
  | "round-answer"    // ввод финальных ответов (20-30 сек), музыка плавно затихает
  | "round-review"    // ведущий проверяет ответы и подводятся итоги раунда

type BlitzPhase =
  | "blitz-captain"   // выбор капитана + выбор порядка игроков (совмещённая фаза)
  | "blitz-pick"      // капитан выбирает задание (получает приватный список со словами)
  | "blitz-ready"     // наушники, музыка включается, все нажимают "Я готов"
  | "blitz-active"    // таймер идёт, музыка играет
  | "blitz-answer"    // ввод финального ответа (20 сек), музыка затихает
  | "blitz-review"    // результат блица
```

### lobby

Главный экран:
- QR-код для подключения игроков
- Подключенные игроки, распределяются по командам. Ведущий может сам перенести игрока в другую команду или кикнуть игрока. 
- кнопка [Калибровка].
- кнопка [Начать]. Доступна, если все игроки готовы.

Экран игрока:
- Игрок вводит имя перед тем, как подключиться в комнату
- Игрок выбирает команду (в командном режиме)
- Игрок выбирает аватарку (из набора эмодзи). Клик по эмодзи = сменить аватар на случайный
- Кнопка [Калиборвка]. Откроет попап калибровки
- Кнопка [Готов]. При нажатии - фиксируется выбор игрока, он больше ничего не может менять.
- Кнопка [Начать] вместо кнопки "Готов", когда все игроки готовы.

### topics-suggest (режим AI)
Игроки предлагают много тем для викторины. Таймер 60с. AI выбирает {N} тем. Затем AI генерирует задания для всех раундов.
Ведущий может самостоятельно указать темы для викторины.

Главный экран (Фаза 1):
- Таймер
- Список всех тем, которые предложили игроки.
- Список N тем, которые выбрал ИИ. Ведущий может удалять и добавлять новые темы.
- Кнопка [Ввести вручную]. Останавливает таймер, ведущий должен ввести темы вручную.
- Кнопка [Продолжить], активна если выбраны {N} тем. Переход к фазе 2

Экран игрока (Фаза 1):
- Текст с просьбой предложить тему для викторины
- Текстовое поле для ввода темы. Каждый игрок может предложить {N} тем (из настроек раунда).
- Кнопка [Больше нет идеи]. Блокирует интерфейс. Если все игроки нажали на кнопку - сразу переход к фазе 2 

Главный экран (Фаза 2):
- Таблица с заданиями, которые сгенерировал ИИ на основе {N} тем.
- Кнопка [Начать игру!]

Экран игрока (Фаза 2):
- Таблица с заданиями, которые сгенерировал ИИ на основе {N} тем.
- Кнопка [Начать игру!]

### round-captain
Игроки выбирают капитана команды. Таймер 60с. Один игрок не может быть капитаном 2 раза подряд.
Если игроки не выбрали капитана, он назначается случайным образом.

Главный экран:
- Таймер
- Таблица с заданиями
- Текст о том, какая команда выбирает капитана.
- Таблица состава активной команды со статусами игроков.

Экран игрока:
- Таймер
- Кнопка [Я капитан]. Кто первый нажал на кнопку, тот становится капитаном.

Экран игрока противоположной команды:
- Таймер
- Текст с пояснением, что команда выбирает капитана.
- Таблица состава активной команды со статусами игроков.

### round-pick
Таймер 60с. Капитан должен выбрать вопрос.
Если вопрос не выбран за отведённое время, выбирается случайный вопрос.

Главный экран:
- Таймер
- Таблица с заданиями. 
- Текст о том, что капитан выбирает вопрос.
- Таблица состава активной команды со статусами игроков.

Экран капитана:
- Таймер
- Текст с просьбой выбрать задание на раунд.
- Таблица с заданиями. Капитан может кликнуть на вопрос и выбрать его
- Карта джокера. Капитан может активировать его.

Экран остальных игроков:
- Таймер
- Таблица с заданиями
- Текст о том, что капитан выбирает вопрос.

### round-ready

Игроки готовятся к раунду.

Главный экран:
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Текст о том, что команда готовится к раунду.
- Таблица состава активной команды со статусами игроков.

Экран активной команды:
- Музыка
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Кнопка [Я готов].

Экран противников:
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Текст о том, что команда готовится к раунду.

### round-active
Капитан видит задание и объясняет его участникам своей команды. Они должны ввести ответ.

Таймер: 50с + 5c * [количество игроков в команде]

Если все игроки ответили на вопросы, переходим к round-review.
Иначе переходим к round-answer.

На главном экране и у активной команды: оповещение звуковым сигналом и вибрацией в начале и конце round-active

Главный экран:
- Таймер
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Таблица состава активной команды со статусами игроков.
- Карточки с ответами игроков. Текст ответа скрыт.

Экран капитана:
- Музыка
- Таймер
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.

Экран игрока:
- Музыка
- Таймер
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Интерфейс, чтобы отправить ответ. Как только игрок ввёл ответ, вместо текстового поля отображается карточка с ответом игрока.

Экран противника:
- Таймер
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.

### round-answer
Игроки, которые не успели ответить, должны ввести ответ.
Таймер 20с.

Главный экран:
- Таймер
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Таблица состава активной команды со статусами игроков.
- Карточки с ответами игроков. Текст ответа скрыт.

Экран капитана:
- Таймер
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Текст: ждём ответа игроков.

Экран игрока, который не успел дать ответ:
- Таймер
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Интерфейс, чтобы отправить ответ. Как только игрок ввёл ответ, вместо текстового поля отображается карточка с ответом игрока.

Экран игрока, который дал ответ:
- Таймер
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- карточка с ответом игрока.

Экран противника:
- Таймер
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.

### round-review

Главный экран (Фаза 1, ручной режим):
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Таблица состава активной команды со статусами игроков.
- Карточки с ответами игроков.
- Для каждой карточки или стопки ведущий отмечает "верно" / "не верно"
- Ведущий может объединять карточки в стопки.
- Бонусное время, если игроки закончили round-active раньше времени.
- Кнопка [Продолжить], переход к фазе 2.

Главный экран (Фаза 1, Режим ИИ):
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Таблица состава активной команды со статусами игроков.
- Карточки с ответами игроков.
- Бонусное время, если игроки закончили round-active раньше времени.
- лоадер с сообщением, что ИИ оценивает ответы игроков.

Главный экран (Фаза 2):
- Таблица с заданиями (уменьшенный вариант)
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Таблица состава активной команды со статусами игроков.
- Карточки или стопки с ответами игроков.
- (режим ИИ) текст с итогом раунда
- Количество баллов, которые заработала команда за раунд. Состояние джокера, если он был использован.
- Кнопка [Оспорить], переход к фазе 1 с ручным режимом.
- Кнопка [Продолжить]

Экран игроков (Фаза 1)
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Текст о том, что ведущий проверяет ответы

Экран игроков (Фаза 2)
- Карточка с заданием. На карточке тема, стоимость задания, текст задания.
- Количество баллов, которые заработала команда за раунд. Состояние джокера, если он был использован.
- У активной команды кнопка [Продолжить]

Баллы за раунд:
Если все ответы верные и количество стопок = количество игроков:
```
[стоимость задания] * [количество игроков] * (1 + ([бонусное время] / [время раунда]))
```
Если хоть один ответ был неверным или объединён, бонусное время не учитывается.
```
[стоимость задания] * [количество стопок с верными ответами]
```
Если активен джокер, то баллы умножаются на 2.

В конце счёт раунда добавляется к общему счёту команды. 

### blitz-captain
Игроки выбирают капитана команды. Таймер 60с. Один игрок не может быть капитаном 2 раза подряд.
После выбора капитана игроки выбирают свою очередь.
Если игроки не успели выбрать капитана или очерёдность, они распределяются случайным образом.

Главный экран:
- Таймер
- Таблица с заданиями
- Текст о том, что команда выбирает капитана и очерёдность.
- Таблица состава активной команды со статусами игроков.

Экран игрока (фаза 1):
- Таймер
- Кнопка [Я капитан]. Кто первый нажал на кнопку, тот становится капитаном.

Экран игрока (фаза 2, после выбора капитана):
- кнопки [1] [2] [3] и т.д. Каждый игрок выбирает слот. Слот бронируется за игроком, который первый на него нажал.

Экран игрока противоположной команды:
- Таймер
- Текст с пояснением, что команда выбирает капитана и очерёдность.
- Таблица состава активной команды со статусами игроков.

### blitz-pick
Таймер 60с.  
Капитан выбирает блиц-задание на раунд.

Главный экран:
- Таймер
- Таблица с заданиями
- Текст о том, что капитан выбирает блиц-задание.
- Таблица состава активной команды со статусами игроков.

Экран капитана:
- Таймер
- Текст с просьбой выбрать блиц-задание на раунд.
- Карточки с блиц-заданиями. На каждой карточке: стоимость, текст (слово или словосочетание). 

Экран остальных игроков:
- Таймер
- Текст о том, что капитан выбирает блиц-задание.

### blitz-ready

Игроки готовятся к блиц-раунду.

Главный экран:
- Таблица с заданиями (уменьшенный вариант)
- Карточка с блиц-заданием. На карточке стоимость задания. Текст задания скрыт.
- Текст о том, что команда готовится к раунду.
- Таблица состава активной команды со статусами игроков.

Экран активной команды:
- Музыка
- Кнопка [Я готов].

Экран противников:
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Текст о том, что команда готовится к раунду.

### blitz-active

Капитан видит блиц-задание и объясняет его участнику 1.
Затем участник 1 объясняет участнику 2. И так до конца очереди.

Таймер: 10с + 20c * ([количество игроков в команде] - 1)

На главном экране и у активной команды: оповещение звуковым сигналом и вибрацией в начале и конце blitz-active

Главный экран:
- Таймер
- Таблица с заданиями (уменьшенный вариант)
- Карточка с блиц-заданием. На карточке стоимость задания. Текст задания скрыт.
- Таблица состава активной команды со статусами игроков.

Экран капитана:
- Музыка
- Таймер
- Карточка с блиц-заданием. На карточке стоимость задания и текст задания.
- Текст: объясните игроку {имя}

Экран последнего игрока:
- Музыка
- Таймер
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Интерфейс, чтобы отправить ответ.
- Текст: Вам объясняет игрок {имя}

Экран промежуточного игрока:
- Музыка
- Таймер
- Карточка с заданием. На карточке тема и стоимость задания. Текст задания скрыт.
- Текст: Вам объясняет игрок {имя}. Затем объясните игроку {имя}

Экран противника:
- Таймер
- Карточка с блиц-заданием. На карточке стоимость задания, текст задания.

Если последний игрок успел ответить на вопрос, переход на blitz-review. Иначе на blitz-answer. 

### blitz-answer
Таймер 20с + 5c * ([количество игроков в команде] - 1)

Игроки, кроме капитана, дают ответы по очереди, начиная с конца.
Если игрок ввёл ответ - переходим к blitz-review.
Если игрок нажал [Не знаю] - очередь переходит к следующему игроку.
Если все игроки нажали [Не знаю] - переходим к blitz-review без ответа.

Экран капитана:
- Таймер
- Текст: ждём ответа игроков.

Экран игрока, который даёт ответ:
- Таймер
- Интерфейс для отправки ответа
- Кнопка [Не знаю]

Экран игрока в очереди:
- Таймер
- Текст: Отвечает игрок {имя}

Экран противника:
- Таймер
- Карточка с заданием. На карточке стоимость задания, текст задания.
- Текст: Отвечает игрок {имя}

### blitz-review

Результаты блиц-раунда

Главный экран:
- Таблица с заданиями (уменьшенный вариант)
- Карточка с блиц-заданием. На карточке стоимость задания, текст задания.
- Таблица состава активной команды со статусами игроков.
- Схема раунда
- Количество баллов, которые заработала команда за раунд.
- Кнопка [Продолжить], переход к фазе 2.

Экран игроков:
- Карточка с блиц-заданием. На карточке стоимость задания, текст задания.
- Количество баллов, которые заработала команда за раунд.
- У активной команды кнопка [Продолжить]

Схема раунда:
Графическое отображение того, как прошёл раунд.
Отображается цепочка игроков, ответ игрока, бонусное время  
Пример: игрок 2 ответил после того, как кончилось время blitz-active.
```text
[Капитан] ➡️ [Игрок 1] ➡️ [Игрок 2] ... ➡️ [Игрок N]
                             ⬇️ (+0:00)              
                        [карточка ответа]
```
Пример: последний игрок ответил за 10с до конца таймера blitz-active.
```text
[Капитан] ➡️ [Игрок 1] ➡️ [Игрок 2] ... ➡️ [Игрок N]
                                              ⬇️ (+0:10)              
                                         [карточка ответа]
```

Баллы за раунд:
Если ответ неверный - 0 баллов.
Если ответ верный, то по формуле  
```
[стоимость задания] * [номер игрока, давшего ответ] * (1 + ([бонусное время] / [время раунда]))
```

В конце счёт раунда добавляется к общему счёту команды.

### finale

Экран игроков:
- Счёт по команд. 
- Текст "Поздравляем с победой!" / "Спасибо за игру!"
- Кнопка [Играем ещё раз!] Доступна в режиме ИИ.

Главный экран:
- Счёт команд
- Таблица заданий
- Статистика игроков:
  - лучший игрок (дал больше всех верных ответов. Не учитывая блиц)
  - лучший капитан (принёс команде больше всех баллов)
  - самый быстрый игрок (в среднем отвечал быстрее всех)
- Кнопка [Играем ещё раз!] Доступна в режиме ИИ.

## Аудио-система

### Файлы
- `assets/music.mp3` — фоновая музыка во время раунда (зацикленная)
- `assets/ring.mp3` — сигнал начала/конца раунда

### Поведение
- Музыка (`music.mp3`) запускается только у **игроков** (не у host и не у spectator) зацикленно
- Запускается автоматически при входе в фазы `round-ready`, `round-active`, `blitz-ready`, `blitz-active`
- Плавное затухание (~3 сек) при переходе в любую другую фазу.
- Калибровка: кнопка "Включить/выключить музыку" позволяет игроку вручную проверить наушники
- Звуковые сигналы (`ring.mp3`): воспроизводятся у **всех** (игроки, host, spectator): в начале и конце фаз `round-active`, `blitz-active`. Сопровождается вибрацией.
- WakeLock API для предотвращения гашения экрана (активируется при входе в игру)
