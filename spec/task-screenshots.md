Задача: сделать скрипт, который будет генерировать скриншоты и сохранять в папку /screenshots.

Пример команды:
`npm run screenshot`

Нужны скриншоты для разных состояний игры.
Для ведущего: в разрешении 1920x1080.
Для игроков: в разрешении 400x700.
Нужно делать скриншоты и в тёмной, и в светлой теме.

Нужно эмулировать определённое состояние игры, без подключения к бекенду.

## Состояния игры для скриншотов

### Главный экран
- Для ведущего и для игрока

### Подключение в комнаду
Для игрока

### lobby
И для ведущего, и для игрока, следующие варианты
- пустой лобби для одной команды
- пустой лобби для двух команд
- заполненный лобби для одной команды, 10 игроков
- заполненный лобби для двух команд по 10 игроков

### calibration
Экран калибровки и для ведущего, и для игрока

### topic-suggest
Для ведущего и для игрока
- 10 игроков в одной команде, нет предложений тем
- 10 игроков в одной команде, 20 предложений тем
- по 10 игроков в каждой команде, нет предложений тем
- по 10 игроков в каждой команде, 20 предложений тем

### question-setup
Для ведущего и для игрока
- Ведущий ещё не загрузил JSON
- Ведущий загрузил JSON

### round-captain
- для ведущего
- для игрока
- для игрока, который уже был капитаном 

С учётом того, что в игре есть загруженные вопросы.

### round-pick
- Для ведущего
- для игрока
- для игрока-капитана

По 3 варианта на каждый из пунктов выше
- джокер не активирован
- джокер активирован на этот раунд
- джокер был активирован ранее 

### round-ready
Для ведущего, для игрока

### round-active
- Для ведущего
- для капитана
- для игрока, который ещё не дал ответ
- для игрока, который уже дал ответ

### round-answer
- Для ведущего
- для игрока, который уже ответил на вопрос в предыдущей фазе
- для игрока, который ещё не ответил на вопрос
- для капитана

### round-review
Для ведущего, и для игрока. Для разных команд (синяя и красная)
Считаем, что 8 игроков дали какой-то ответ, 2 игрока не успели.

### round-result
Для ведущего, и для игрока.
Считаем, что 8 игроков дали ответ, из которых 4 верных. 2 вопроса объединили (по синонимам). 2 игрока не успели.

### blitz-captain
- для ведущего
- для игрока
- для игрока, который уже был капитаном 

С учётом того, что в игре есть загруженные задания.

### blitz-pick
Для ведущего, для игрока, для игрока-капитана

### blitz-ready
Для ведущего, для игрока

### blitz-active
- Для ведущего
- для капитана
- для игрока, который ещё не дал ответ
- для игрока, который уже дал ответ

### blitz-answer
- Для ведущего
- для игрока, который уже ответил на задание в предыдущей фазе
- для игрока, который ещё не ответил на задание
- для капитана

### blitz-result
- для игрока
- для ведущего

Считаем, что 5 игроков дали правильный ответ, 4 игрока - неправильный, 1 игрок не успел ответить.

### finale
- Для ведущего, если участвовала только 1 команда
- Для ведущего, если было 2 команды
- для игрока, если участвовала только 1 команда
- Для игрока победившей команды, если было 2 команды
- Для игрока проигравшей команды, если было 2 команды

## Дополнительно
- Скрипт должен генерировать скриншоты с уникальными именами в папке /screenshots
- Скрипт выводит список скриншотов, а также информацию: получилось или не получилось сделать скриншот.
- Скрипт выводит тексты ошибок после списка скриншотов
- Раздели функционал добавления скриншотов на отдельные файлы. В шапке пиши комментарий, какие скриншоты генерирует этот файл.
