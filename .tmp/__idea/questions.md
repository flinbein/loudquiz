# Задание для обычного раунда

## Правила составления вопросов

- Вопросы должны быть разной сложности, в оценке от 100 баллов (легко) до 200 баллов (очень сложно)

- При составлении вопроса нужно учитывать, что игроки не слышат друг друга, и капитан будет объяснять вопрос мимикой и жестами. Игроки затем должны договориться между собой, чтобы не написать одинаковые ответы. На сложность вопроса влияет то, насколько сложно передать смысл вопроса другим игрокам с помощью жестов, а также сложность самого вопроса. Длинная формулировка вопроса увеличивает его сложность.

- Вопрос должен иметь несколько уникальных вариантов ответов (не менее, чем количество игроков в команде)

- Ведущий должен иметь возможность проверить правильность ответа.

- Для увеличения сложности, можно добавлять дополнительные исключающие условия. Например:
  - День недели, не начинающийся на букву "П" (исключить понедельник, пятница)
  - месяц, содержащий 31 день
  - Млекопитающее из трёх слогов
  - Мультфильм, вышедший после 2005г (исключить старые мультфильмы)
  - Музыкальный инструмент, не состоящий из дерева (исключить инструменты, содержащие дерево)

- Для увеличения сложности, вопрос может содержать каламбур или загадку. Примеры:
  - Город в стране восходящего солнца. (вместо "Город в Японии")
  - Персонаж из фильма, одноименного с названием двумерного массива данных (вместо "Персонаж из фильма Матрица")

## Оценка ответов

Ведущий оценивает ответы игроков по следующим критериям:
- ответ должен быть правильным. Ведущий может убедиться, что ответ верный.
  > Человек, который летал в космос
  > * Я

  Если ведущий не может проверить правильность ответа, то ответ считается неверным.

- ответ не должен быть расплывчатым. Если игрок плохо понял вопрос и решил сжульничать, такой ответ не засчитывается. Пример расплывчатого ответа:
  > Фрукт, который растёт в Африке
  > * Африканский фрукт

- ответ должен быть уникальным для текущего раунда. Если 2 игрока ответили одинаково, то ведущий объединяет их в 1 ответ. Одинаковыми считаются в том числа ответы, похожие по смыслу. Пример одинаковых ответов:
  > Человек, который летал в космос
  > * Гагарин
  > * Юрий Гагарин

  За исключением вопросов, в которых допускается использовать похожие по смыслу слова. В этом случае в зачёт идут оба ответа. Например:
  > Синоним слова "Смелый"
  > * Бесстрашный (принимается)
  > * Отважный (принимается)

- В ответах допускаются небольшие опечатки или орфографические ошибки, если в вопросе не предусматривается строгое написание слова. Например
  > Любое слово, содержащее "ЦЫ"
  > * Цыган (принимается)
  > * Цыплёнок (принимается)
  > * Цыфра (не принимается)

Любые спорные моменты ведущий решает на своё усмотрение. При наличии спорных моментов ведущий разъясняет свою точку зрения и почему принял такое решение.

# Задания для блиц-раунда

## Правила составления заданий

На каждый раунд нужно подготовить {N} заданий.

Задание - это слово или словосочетание, которое один игрок должен будет объяснить другому с помощью жестов и мимики.

Задания должны быть разной сложности, в оценке от 200 баллов (легко) до 400 баллов (очень сложно). Примерно равное распределение по сложности.

Пример заданий на раунд:
- Коза (200, просто показать жестами козу, легко прочитать по мимике)
- Трансформатор (270, узкоспециализированное слово. Длинное, сложно передать мимикой)
- Маленький принц (350, словосочетание)
- мост в Терабитию (400, словосочетание с предлогом)


Сложность задания определяется следующими правилами:
- Чем сложнее передать слово с помощью жестов и мимики, тем выше сложность
- Узкоспециализированное слово = повышенная сложность.
- Словосочетание = повышенная сложность.
- Словосочетание с предлогом = ещё сложнее.

## Проверка ответа

Ответ должен совпадать с загаданным словом или словосочетанием.
- Проверяется без учёта регистра
- Игнорируются спецсимволы и пробелы `крем-брюле` = `крем брюле`= `крембрюле`
- `е` или `ё` - не имеет значения
