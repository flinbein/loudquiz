# Редактор вопросов

Веб-интерфейс для подготовки JSON-файла с вопросами.
Доступен по маршруту `/constructor`.

## Формат файла

```typescript
interface QuestionsFile {
  topics: Topic[];
  blitzTasks?: BlitzTask[];
}

interface Topic {
  name: string;
  questions: Question[];
}

interface Question {
  text: string;
  difficulty: number;         // 100–200, кратно 10
  acceptedAnswers?: string[]; // примеры верных ответов (заполняется AI)
}

interface BlitzTask {
  id: string;
  items: BlitzItem[];         // 3–5 вариантов разной сложности
}

interface BlitzItem {
  text: string;               // слово/словосочетание (задание = ответ)
  difficulty: number;         // 200–400, кратно 10
}
```

## Ручной режим

Пользователь заполняет формы вручную:
- Создаёт темы, вводит названия.
- Для каждой темы добавляет вопросы: текст + сложность.
- Добавляет блиц-задания: текст + сложность.

Результат экспортируется как JSON-файл.

## AI-режим

Для работы с AI пользователь указывает **OPENROUTER_API_KEY**
(сохраняется в localStorage).

### Генерация тем

1. Пользователь вводит предложения тем (многострочный ввод).
2. Указывает количество тем N.
3. AI выбирает N тем и объясняет выбор.
4. Пользователь нажимает **[Применить]** — создаются формы Topic.

Подробнее: [Генерация тем AI](../systems/ai/topic-generation.md).

### Генерация вопросов

1. Имеются заполненные формы Topic (с названиями тем).
2. Пользователь указывает количество игроков X и вопросов N.
3. AI генерирует N вопросов для каждой темы с `acceptedAnswers`.
4. Пользователь нажимает **[Применить]** — формы заполняются вопросами.

Подробнее: [Генерация вопросов AI](../systems/ai/question-generation.md).

### Проверка вопросов

Около каждого вопроса — кнопка **«Проверить ответы»**. Открывает попап:

1. Поле с ответами (предзаполнено из `acceptedAnswers`).
2. Кнопка **«Проверить»**.
3. AI проверяет ответы и отображает результат:
   - Какие ответы верные.
   - Какие объединены и почему.
   - Краткий итог.

Подробнее: [Проверка ответов AI](../systems/ai/answer-check.md).

### Генерация блиц-заданий

1. Пользователь указывает количество блиц-раундов X и заданий N.
2. Указывает список прошлых блиц-заданий (хранится в localStorage).
3. AI генерирует задания.
4. Пользователь нажимает **[Применить]** — заполняется форма блиц-заданий.

Подробнее: [Генерация блиц-заданий AI](../systems/ai/blitz-generation.md).
