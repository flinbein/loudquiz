# Редактор вопросов

Редактор вопросов позволяет подготовить JSON файл с вопросами для игры.

Формат файла:
```typescript
interface QuestionsFile {
  topics: Topic[];
  blitzTasks?: BlitzTask[];
}

interface Topic {
  name: string;              
  questions: Question[];
}

// Задание на обычный раунд
interface Question {
  text: string;
  difficulty: number;         // от 100 до 200, кратно 10 — задаётся вручную в редакторе
  acceptedAnswers?: string[]; // пример ответов, которые ИИ посчитал бы верным. Не заполняется в ретакторе
}

// BlitzTask — один раунд блица. Содержит набор вариантов слов/словосочетаний
// разной сложности. Капитан объясняет слова жестами, команда называет их вслух.
interface BlitzItem {
  text: string;       // слово или словосочетание — задание и ответ одновременно
  difficulty: number; // 200..400, кратно 10
}

interface BlitzTask {
  id: string;
  items: BlitzItem[]; // обычно 3–5 вариантов разной сложности
}
```

Пользователь может вписывать ответы вручную, а может воспользоваться ИИ.

## Генерация вопросов с помощью ИИ

Для работы с ИИ юзер должен указать OPENROUTER_API_KEY. Он сохраняется в localStorage.

### Генерация тем

Юзер заполняет:
- Предложения тем для викторины. Многострочный ввод.
- Количество тем {N}, которые должен выбрать ИИ.

ИИ генерирует:
- Список тем {N} штук, которые он выбрал для викторины.
- Объясняет, почему выбрал именно эти темы.

Потом юзер жмёт [Применить], и создаются формы `Topic`, с заполненным `name`.
Юзер может далее заполнять формы вручную, или же сгенерировать вопросы с помощью ИИ

### Генерация вопросов

У юзера есть несколько форм `Topic`, учитываются названия тем.

Юзер заполняет:
- Количество игроков {X}, которые будут участвовать в викторине.
- Количество вопросов {N} в каждой теме.

ИИ генерирует:
- {N} вопросов разной сложности для каждой темы, учитывая, для какого количества игроков предполагается викторина.
- acceptedAnswers - какие ответы, по мнению ИИ, посчитались бы верными.

Потом юзер жмёт [Применить], и формы `Topic` заполняются вопросами.

### Проверка вопросов
Около каждого вопроса кнопка "Проверить ответы". Открывается попап.

Юзер заполняет:
- Ответы игроков (textarea). Поле предзаполнено значением acceptedAnswers

Юзер жмёт кнопку "Проверить".

ИИ проверяет список ответов.

Юзеру отображается информация в попапе:
- Какие ответы верные
- Какие ответы были сгруппированы и почему
- Краткий итог раунда.

### Генерация блиц-заданий

Юзер заполняет:
- Количество блиц-раундов {X}.
- Количество заданий в каждом раунде {N}.
- Список блиц-заданий, которые уже участвовали в игре (хранится в localStorage)

ИИ генерирует:
- {N} заданий разной сложности на каждый из {X} раундов.

Потом юзер жмёт [Применить], заполняется форма с блиц-заданиями.
